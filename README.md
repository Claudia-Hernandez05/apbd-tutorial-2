###### \## Answers

###### 

###### \### 1. Difference between fast-forward merge and merge with commit

###### A fast-forward merge happens when the target branch has no new commits, so Git only moves the branch pointer forward.

###### A merge with commit creates a new merge commit and keeps the history of both branches.

###### 

###### \### 2. Difference between merge and rebase

###### Merge combines the histories of two branches and preserves the branch structure.

###### Rebase rewrites the branch history by placing its commits on top of another branch, creating a cleaner linear history.

###### 

###### \### 3. How the conflict was resolved

###### The conflict was created by editing the same file in `main` and `conflict-branch`.

###### It was resolved manually by opening `conflict.txt`, keeping both changes, saving the file, and then committing the resolved version.
